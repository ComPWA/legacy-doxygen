193  concurrent_vector_base_v3() {
194  //Here the semantic is intentionally relaxed.
195  //The reason this is next:
196  //Object that is in middle of construction (i.e. its constructor is not yet finished)
197  //cannot be used concurrently until the construction is finished.
198  //Thus to flag other threads that construction is finished, some synchronization with
199  //acquire-release semantic should be done by the (external) code that uses the vector.
