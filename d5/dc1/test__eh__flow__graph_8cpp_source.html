116 // for nodes with limited concurrency, if that concurrency is < g_NumThreads, we need
117 // to make sure enough other nodes wait for concurrency to peak. If we are attached to
118 // N successors, for each item we pass to a successor, we will get N executions of the
119 // "absorbers" (because we broadcast to successors.) for an odd number of threads we
120 // need (g_NumThreads - limited + 1) / 2 items (that will give us one extra execution
121 // of an "absorber", but we can't change that without changing the behavior of the node.)

780 //
781 // Continue node has T predecessors. when it receives messages (continue_msg) on T predecessors
782 // it executes the body of the node, and forwards a continue_msg to its successors.
783 // However many predecessors the continue_node has, that's how many continue_msgs it receives
784 // on input before forwarding a message.
785 //
786 // The graph will look like
787 //
788 // +broadcast_node+
789 // / \ ___
790 // source_node+------>+broadcast_node+ +continue_node+--->+absorber
791 // \ /
792 // +broadcast_node+
793 //
794 // The continue_node has unlimited parallelism, no input buffering, and broadcasts to successors.
795 // The absorber is parallel, so each item emitted by the source will result in one thread
796 // spinning. So for N threads we pass N-1 continue_messages, then spin wait and then throw if
797 // we are allowed to.
798 
799 template < class SourceNodeType, class SourceNodeBodyType, class TTestNodeType, class TestNodeBodyType,
800  class SinkNodeType, class SinkNodeBodyType>

810  o.observe(true);
811 #endif
